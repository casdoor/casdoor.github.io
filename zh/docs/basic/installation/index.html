安装教程

B. 所需经费​

  • Go 1.6+
  • Node.js LTS (16或14)
  • Yarn 1.x

下载​

城门源代码托管在 GitHub: https://github.com/casbin/casdoor 。 转到后端代码和 React 前端代码都在单个仓库中。

城门支持 转到模块。 要下载代码，你只能克隆代码到任何目录：

cd 路径/to/文件夹
git clone https://github.com/casbin/casdoor

配置​

所有配置都放置在 conf/中，包括 app.conf 和 oss.conf 中。 前者储存与应用程序有关的信息，而后者只储存与OSS有关的配置。 您可以修改两个文件来设置所有配置。

但第一次使用时，你不需要找出所有的东西，这些东西将在以后引入。 现在你只需要几个简单的配置来在你的个人电脑上运行连锁门。

然而，仍然存在着这种情况。 数据库信息应该首先配置(除非您选择使用特殊的图像连铸/个案门-所有内部，专门为新用户进行试用)。 详情请参阅章节运行/停靠符号)

Cassdoor 使用 XORM 框架连接到DB, 所以所有由 XORM 支持的DB也可以被使用。

  • 设置数据库 (MySQL)：

    Casdoor 把用户、节点还有主题信息保存在MySQL数据库的casdoor库中，如果这个数据库不存在就会自动创建一个。 数据库连接字符串可以在 https://github.com/casbin/casdoor/blob/master/conf/app.conf 指定。

    驱动器名称 = mysql
    datasSourceName = root:123456@tcp(localhost:3306)/
    dbName = catdoor

  • 设置数据库 (Postgres)：

    由于在使用xorm打开 Postgres 时必须选中一个数据库，所以你应该在启动 Casdoor 前手动准备好数据库。

    假设你已经创建了一个叫做 casdoor 的数据库，然后你应该把 app.conf 修改为这样：

    驱动器名称 = postgres
    dataSourceName = "user=postgres password=xxx sslmode=disable dbname="
    dbName = casdoor

    注意： 你可以通过 dataSourceName 添加 Postgres 参数，但是请确保 dataSourceName 以 dbname= 结尾。 否则数据库适配器可能在你启动 Casdoor 的时候崩溃。

  • 其它数据库...

运行​

当前有两种方法可以启动，您可以根据自己的情况选择一个。

手动安装​

根据实际需要，它被分为 dev 模式和 prod 模式

开发环境​

完成上述简单配置步骤后，您可以运行 Casdoor。 城门有一个前端后端分离结构。 前端使用 javascript 和 React，后端使用 beego 和 xorm

后端​

城堡后端默认在 8000 端口运行。 您可以通过以下命令启动后端：

前往运行 main.go

服务器运行成功后，我们可以启动前端部件。

前端​

Cassdoor的React 前端默认在端口 7001。 城门使用 Yarn 1.x 作为官方包管理器。 打开一个新终端并运行：

cd web
yarn installing
yarn starn

:::注意事项

我们强烈建议您使用 Yarn 1。 若要安装Casdoor，使用NPM可能会导致UI风格问题，详情见： casdoor#294

:::

打开您最喜欢的浏览器并访问 http://localhost:7001，您将看到Cassdoor的登录页面。

输入 admin and 123 测试登录功能正常工作。

生产部署​

前端​

构建静态页面：

yarn building

后端​

我们将转到后端代码构建为可执行文件: 种子门 然后启动它:

去构建 && ./casdoor

接下来，请访问： http://localhost:8000 在您的浏览器中，如果您可以看到Cassdoor的登录页面， 并且您可以使用 管理员 和 123 作为用户名和密码登录，然后一切都是肯定的！

要使用另一个端口, 请编辑 conf/app.conf 然后修改 http端口, 然后重启去后端。 :::

在 dev 环境中，前端由 yarn 运行 端口7001 所以，如果您想去Casto登录页面, 您需要设置Casdoor 链接为 http://localhost:7001。

在 prod 环境中，前端文件首先由 yarn building 构建并服务于端口 8000， 如果您想要转到连锁店登录页面， 您需要设置 Casdoor 链接为 http://SERVER_IP:8000 (如果您正在使用逆向代理) 您需要将链接设置为 您的域

以我们的官方论坛 Casnode 作为示例：​

Casnode 使用 Casdoor 来处理身份验证。

当我们在 dev 环境中测试Casnode 时，我们将 serverUrl 设置为 http://localhost:7001。 所以当我们使用 Cassdoor 测试登录和注册功能时，它将会转到本地主机7001，而本地端口是Casdoor 端口。

当我们将Casnode 到 prod 环境时，我们将 serverUrl 设置为 https://door。 asbin.com，用户可以使用Casdoor登录或注册。

修改配置​

为了方便您的第一次尝试，docker-compose.yml 包含了通过docker启动数据库的命令。

因此，编辑 conf/app.conf 来指出数据库位置(db:3306)，修改 数据源名称 到固定内容：

dataSourceName = root:123456@tcp(db:3306)/

如果您需要修改 conf/app.conf，您需要重新运行 docker-compose up。

一些详细信息​

  • Github corner

    我们在右上角添加了一个 Github 图标，链接到您的 Github 仓库地址。 您可以设置 ShowGithubCorner 来隐藏它。

    配置：

    导出 const ShowGithubCorner = true

    导出 const GithubRepo = "https://github.com/casbin/casdoor" ///your github 仓库

  • OSS conf

    我们使用开放源码软件储存和提供用户头像。 您必须修改文件 conf/oss.conf 来告诉您的 OSS 信息后端。 对于开放源码软件供应商，我们现在支持Aliyun([aliyun]), awss3([s3])。

    [provider]
    accessId = id
    accessKey = key
    bucket = bucket
    endpoint = endpoint

    请正确填写此conf ，否则头像服务器将无法工作！

  • 应用conf

    conf/app.conf 存储重要的腰果配置，这对您更好地使用城门非常有用。

    appname = casdoor
    httpport= 8000
    runmode = dev
    Sessionon = true
    copyrequestbody = true
    driverName = mysql
    dataSourceName = root:123@tcp(localhost:3306)/
    dbname = casdoor
    State = "casdoor"
    useProxy = false
    VerificationCodeTimeout = 10
    • appname 是应用程序名称，目前没有实际使用
    • http端口 是您后端应用程序正在监听的端口
    • 运行模式 是 dev 或 prod
    • 会话 决定是否启用会话，默认使用。
    • 驱动名称, 数据源名称 和 dbname 此前已启用，请参阅 配置。
    • 使用 Proxy 设置代理端口，因为我们有谷歌相关的服务，这些服务将在某些地区受到网络限制
    • 验证 CodeTimeout 设置验证码到期时间。 到期后，用户需要再次获取

  • 如果户需要公开访问(拥有公共IP)，也需要修改 web/src/auth/Auth.js：

    导出 authConfig =
      serverUrl: "http://example.com", // 你的城门URL, 就像官方网址: https://door.casbin.com, 或 http:1.1.1:8000
    ...
    }
