サーバーのインストール

要件​

OS​

すべてのOS(Linux、Windows、macOS)がサポートされています。

環境​

  • Go 1.6+
  • Node.js LTS (16または14)
  • Yarn 1.x
info

We strongly suggest you use Yarn 1.x to run & build Casdoor frontend, using NPM might cause UI styling issues, see more details at: casdoor#294

データベース​

Casdoor は XORM をデータベースと通信するために使用します。 Xormドライバのサポートに基づいて、現在サポートされているデータベースは次のとおりです。

  • MySQL
  • MariaDB
  • PostgreSQL
  • SQL Server
  • Oracle
  • SQLite 3
  • TiDB

ダウンロード​

CasdoorのソースコードはGitHubでホストされています: https://github.com/casdoor/casdoor . Go バックエンドコードと React のフロントエンドコードはどちらも単一のリポジトリ内にあります。

名前説明言語ソース コード
FrontendCasdoor用WebフロントエンドUIJavaScript + Reacthttps://github.com/casdoor/casdoor/tree/master/web
バックエンドCasdoor の RESTful API バックエンドです。Golang + Beego + XORMhttps://github.com/casdoor/casdoor

Casdoor は Go モジュール をサポートしています。 コードをダウンロードするには、Git経由でコードをクローンするだけです。

cd path/to/folder
git clone https://github.com/casdoor/casdoor

設定​

Casdoor は 1 つのファイルで構成できます: conf/app.conf。デフォルトでは以下の内容です:

appname = casdoor
httpport = 8000
runmode = dev
SessionOn = true
copyrequestbody = true
driverName = mysql
dataSourceName = root:123456@tcp(localhost:3306)/
dbName = casdoor
redisEndpoint =
defaultStorageProvider =
isCloudIntranet = false
authState = "casdoor"
httpProxy = "127.0.0.1:10808"
verificationCodeTimeout = 10
initScore = 2000
logPostOnly = true
origin = "https://door.casbin.com"
  • appname は現在実用化されていないアプリケーション名です。
  • httpport はバックエンドアプリケーションが待ち受けているポートです
  • runmode は dev または prod
  • SessionOn がセッションを有効にするかどうかを決定します。デフォルトで使用されます。
  • driverName, dataSourceName と dbName は以前に導入されています。 Config を参照してください。
  • useProxy はプロキシポートを設定します。Googleに関連するサービスがあるため、一部の地域ではネットワークによって制限されます。
  • verificationCodeTimeout は、検証コードの有効期限を設定します。 有効期限が切れた後、再度取得する必要があります

すべての設定可能なフィールドにもかかわらず、初心者として。 データベースに基づいて、 driverName と dataSourceName の2つの項目を変更するだけです。 このデータベースは、Casdoorがユーザー、組織、アプリケーションなどのすべてのデータを保存するために使用されます。

  • データベースのセットアップ (MySQL):

    Casdoorは、ユーザー、ノード、トピック情報をMySQLデータベースに格納します: casdoorが存在しない場合、それを作成します。 DB 接続文字列は、https://github.com/casdoor/casdoor/blob/master/conf/app.conf で指定できます。

    driverName = mysql
    dataSourceName = root:123456@tcp(localhost:3306)/
    dbName = casdoor

  • データベースのセットアップ (Postgres):

    xormでPostgresを開くときにはデータベースを選択する必要があるので、Casdoorを実行する前に手動でデータベースを準備する必要があります。

    既に casdoorという名前のデータベースが用意されていると仮定し、 app.conf を次のように指定する必要があります。

    driverName = postgres
    dataSourceName = "user=postgres password=xxx sslmode=disable dbname="
    dbName = casdoor

    Please notice: You can add Postgres parameters in dataSourceName, but please make sure that dataSourceName ends with dbname=. または、Casdoorを起動するとデータベースアダプタがクラッシュする可能性があります。

  • 他のデータベース...

実行​

現在、始めるには2つの方法があり、自分の状況に応じて1つを選択することができます。

開発モード​

バックエンド​

CasdoorのGoバックエンドはデフォルトで8000ポートで動作します。 Go バックエンドを開始するには、次のコマンドを使用します。

行けmain.go

サーバーが正常に実行された後、フロントエンド部分を開始できます。

Frontend​

Casdoorのフロントエンドは非常に古典的な Create-React-App (CRA) プロジェクトです。 デフォルトではポート 7001 で動作します。 次のコマンドを使用してフロントエンドを実行します。

cd web
yarn install
yarn start

訪問: http://localhost:7001 をブラウザで確認してください。 デフォルトのグローバル管理者アカウントでCasdoorダッシュボードにログインします: 組み込み/管理

管理者
123

プロダクションモード​

バックエンド​

ビルドCasdoor バックエンドコードを実行可能にして起動します。

Linux の場合:


./casdoor を作ろう

Windowsの場合:


casdoor.exe をビルドする

Frontend​

Casdoor フロントエンドコードを静的リソース (.html、.js、.css ファイル) に構築します。

cd web
yarn install
yarn build

訪問: http://localhost:8000 をブラウザに表示します。 デフォルトのグローバル管理者アカウントでCasdoorダッシュボードにログインします: 組み込み/管理

管理者
123
別のポートを使用するには、 conf/app.conf を編集して httpportを変更し、Go backend を再起動してください。 :::

In dev environment, the frontend is run by yarn run in port 7001, so if you want to go to Casdoor login page, you need set Casdoor link as http://localhost:7001.

In prod environment, the frontend files is first built by yarn build and served in port 8000, so if you want to go to Casdoor login page, you need to set Casdoor link as http://SERVER_IP:8000 (If you are using reverse proxy, you need to set the link as your domain).

公式フォーラムCasnodeを例として取りましょう。​

Casnode は認証を処理するために Casdoor を使用します。

dev 環境でCasnodeをテストしているときは、 serverUrl を http://localhost:7001として設定します。 Casdoorを使用してサインインとサインアップ機能をテストすると、Casdoorポートであるlocalhost7001に移行します。

そしてCasnodeを prod 環境にすると、 serverUrl をhttps://doorとして設定します。 asbin.comは、ユーザーがCasdoorを使用してサインインまたはサインアップすることができます。
