スプリングブーツ

casdoor-spring-boot-example は SpringBoot プロジェクトで casdoor-java-sdk を使用する方法の例です。 以下の手順をお見せします。

Step1. Casdoor を展開​

まず、Casdoorを展開する必要があります。

サーバーインストール については、Casdoor の公式ドキュメントを参照してください。

デプロイが完了したら、以下を確認する必要があります。

  • Casdoor サーバーは http://localhost:8000 で正常に実行されています。
  • お気に入りのブラウザを開き、 http://localhost:7001を参照してください。Casdoorのログインページが表示されます。
  • ログイン機能をテストするには、 管理者 と 123 を入力してください。

次に、以下の手順で、キャスドアベースのログインページを独自のアプリにすばやく実装できます。

Step2. casdoor-java-sdk をインポート​

casdoor-java-sdkは、mavenまたはgradleでインポートできます。

  • Maven
  • Gradle
<dependency>
    <groupId>org.casbin</groupId>
    <artifactId>casdoor-java-sdk</artifactId>
    <version>1.x.y</version>
</dependency>
implementation 'org.casbin:casdoor-java-sdk:1.x.y'

ステップ3. 初期化設定​

初期化には6つのパラメータが必要です。これはすべて文字列型です。 | 名前（順番） | 必須項目 | 説明 | | ---------------- | ---- | ----------------------------------------------- | | endpoint | はい | Casdoor Server Url ( http://localhost:8000 など | | clientId | はい | アプリケーション.client_id | | clientSecret | はい | Application.client_secret | | jwtSecret | はい | Casdoor JWTの秘密と同じです。 | | organizationName | はい | Application.組織 | | applicationName | はい | Application.name | Java プロパティまたは YAML ファイルを使用して、以下のように初期化できます。

  • Properties
  • YML
casdoor.endpoint = http://localhost:8000
casdoor.clientId = 874e3e05e58d50148c65
casdoor.clientSecret = 41510b84c7267ad2e4d2b51096b7f11dc9c5fdc8
casdoor.jwtSecret = CasdoorSecret
casdoor.organizationName = built-in
casdoor.applicationName = app-built-in
casdoor:
  endpoint: http://localhost:8000
  client-id: 874e3e05e58d50148c65
  client-secret: 41510b84c7267ad2e4d2b51096b7f11dc9c5fdc8
  jwt-secret: CasdoorSecret
  organization-name: built-in
  application-name: app-built-in

:::注意 特に clientId と clientSecret を独自のCasdoorインスタンスに置き換える必要があります。 ::: 次に、 CasdoorSdkProperties クラスを作成し、設定に対応するメンバー変数を宣言し、getter と setter を提供します。

@Data
@Component
@ConfigurationProperties(prefix = "casdoor")
public class CasdoorSdkProperties {
    private String endpoint;
    private String clientId;
    private String clientSecret;
    private String jwtSecret;
    private String organizationName;
    private String applicationName;
}

CasdoorSdkConfig クラスを作成し、CasdoorSdkPropertiesのインスタンスを使用して CasdoorConfig を初期化します。

@Configuration
public class CasdoorSdkConfig {

    @Resource
    private CasdoorSdkProperties casdoorSdkProperties;

    @Bean
    public CasdoorConfig getCasdoorConfig() {
        return new CasdoorConfig(
                casdoorSdkProperties.getEndpoint(),
                casdoorSdkProperties.getClientId(),
                casdoorSdkProperties.getClientSecret(),
                casdoorSdkProperties.getJwtSecret(),
                casdoorSdkProperties.getOrganizationName(),
                casdoorSdkProperties.getApplicationName()
        );
    }
}

SpringBoot Application が起動すると、Java プロパティまたは YAML の構成が自動的に CasdoorSdkProperties クラスのメンバー変数に注入されます。

そのため、 CasdoorSdkConfig クラスの getCasdoorConfig メソッドを使用して、どこでも CasdoorConfig インスタンスを取得できます。

Step4. ログインページにリダイレクト​

アプリにアクセスする認証が必要な場合は、ターゲットURLを送信し、Casdoorが提供するログインページにリダイレクトすることができます。

事前にアプリケーションの設定にコールバックURLを追加していることを確認してください (例: http://localhost:8080/login)。

@RequestMapping("toLogin")
public String toLogin() {
    CasdoorConfig casdoorConfig = casdoorSdkConfig.getCasdoorConfig();
    String targetUrl = String.format("%s/login/oauth/authorize?client_id=%s&response_type=code&redirect_uri=%s&scope=read&state=%s",
            "http://localhost:7001", casdoorConfig.getClientId(),
            "http://localhost:8080/login", casdoorConfig.getApplicationName());
    return "redirect:" + targetUrl;
}

ステップ5. トークンを取得して解析する​

Casdoorの検証が完了すると、コードと状態でアプリケーションにリダイレクトされます。

コードを取得して getOAuthToken メソッドを呼び出して、jwtトークンを解析することができます。

CasdoorUser にはCasdoorが提供するユーザーに関する基本情報が含まれています。 キーワードとしてアプリケーションでセッションを設定できます。

@RequestMapping("login")
public String login(String code, String state, HttpServletRequest request) {
    CasdoorAuthService casdoorAuthService = new CasdoorAuthService(casdoorSdkConfig.getCasdoorConfig());
    String token = "";
    CasdoorUser user = null;
    try {
        token = casdoorAuthService.getOAuthToken(code, state);
        user = casdoorAuthService.parseJwtToken(token);
    } catch (OAuthSystemException | OAuthProblemException | ParseException | InvocationTargetException | IllegalAccessException e) {
        e.printStackTrace();
    }
    HttpSession session = request.getSession();
    session.setAttribute("casdoorUser", user);
    return "redirect:/";
}

Step6. UserService​

CasdoorUserService は以下のような基本的なユーザー操作をサポートしています。

  • GetUser(name string), ユーザ名で1人のユーザを取得する.
  • GetUsers(), すべてのユーザーを取得します。
  • UpdateUser(auth.User)/AddUser(auth.User)/DeleteUser(auth.User), データベースにユーザを書き込みます.
