Serverinstallation

Anforderungen​

OS​

Alle Betriebssysteme (Linux, Windows und macOS) werden unterstützt.

Umgebung​

  • Go 1.6+
  • Node.js LTS (16 oder 14)
  • Yarn 1.x

:info

Wir empfehlen dringend, Yarn 1 zu verwenden. zum Ausführen von & Build Casdoor Frontend, mit NPM könnte UI Stylingprobleme verursachen siehe mehr Details auf: casdoor#294

:::

Datenbank​

Casdoor verwendet XORM um mit der Datenbank zu sprechen. Basierend auf Xorm Drivers Support, Die aktuellen unterstützten Datenbanken sind:

  • MySQL
  • MariaDB
  • PostgreSQL
  • SQL-Server
  • Jürgen
  • SQLite 3
  • TiDB

Download​

Der Quellcode von Casdoor befindet sich auf GitHub: https://github.com/casdoor/casdoor . Sowohl der Go-Backend Code als auch der React Frontend Code befinden sich innerhalb des Single Repository.

NameBeschreibungSpracheQuellcode
FrontendWeb-Frontend-UI für CasdoorJavaScript + Reaktionhttps://de.github.com/casdoor/casdoor/tree/master/web
BackendRESTful API Backend für CasdoorGolang + Beego + XORMhttps://de.github.com/casdoor/casdoor

Casdoor unterstützt Go-Module. Um den Code herunterzuladen, können Sie einfach den Code per git klonen:

cd path/to/folder
git clone https://github.com/casdoor/casdoor

Konfiguration​

Casdoor kann über eine einzelne Datei konfiguriert werden: conf/app.conf, dessen Inhalt standardmäßig ist:

appname = casdoor
httpport = 8000
runmode = dev
SessionOn = true
copyrequestbody = true
driverName = mysql
dataSourceName = root:123456@tcp(localhost:3306)/
dbName = casdoor
redisEndpoint =
defaultStorageProvider =
isCloudIntranet = false
authState = "casdoor"
httpProxy = "127. .0.1:10808"
verificationCodeTimeout = 10
initScore = 2000
logPostOnly = true
origin = "https://door.casbin.com"
  • appname ist der Name der Anwendung, der derzeit keine praktische Verwendung hat
  • httpport ist der Port, an dem Ihre Backend-Anwendung lauscht
  • runmode ist dev oder prod
  • SessionOn entscheidet, ob Session aktiviert werden soll. Standardmäßig verwendet wird.
  • TreiberName, dataSourceName und dbName werden bereits eingeführt, bitte lesen Sie Konfiguration.
  • useProxy setzt den Proxy-Port, da wir Google-bezogene Dienste haben, die in einigen Bereichen durch das Netzwerk eingeschränkt werden
  • VerificationCodeTimeout hat die Ablaufzeit des Bestätigungscodes festgelegt. Nach Ablauf muss der Benutzer es erneut erhalten

Trotz aller konfigurierbaren Felder als Anfänger Sie müssen nur zwei Elemente ändern: driverName und dataSourceName basierend auf Ihrer Datenbank. Diese Datenbank wird von Casdoor verwendet, um alle Daten, einschließlich Benutzer, Organisationen, Anwendungen usw. zu speichern.

  • Datenbank (MySQL):

    Casdoor speichert seine Benutzer, Knoten und Themen in einer MySQL-Datenbank: casdoor, wird sie erstellen, wenn sie nicht existiert. Der DB-Verbindungsstring kann unter https://github.com/casdoor/blob/master/conf/app.conf angegeben werden

    driverName = mysql
    dataSourceName = root:123456@tcp(localhost:3306)/
    dbName = casdoor

  • Datenbank einrichten (Postgres):

    Da wir beim Öffnen von Postgres mit xorm eine Datenbank wählen müssen, sollten Sie eine Datenbank manuell vorbereiten, bevor Sie Casdoor ausführen.

    Nehmen wir an, Sie haben bereits eine Datenbank mit dem Namen casdoorvorbereitet, dann sollten Sie app.conf so angeben:

    driverName = postgres
    dataSourceName = "user=postgres password=xxx sslmode=deaktivieren dbname="
    dbName = casdoor

    Bitte beachten: Sie können Postgres-Parameter in dataSourceName hinzufügen, aber bitte stellen Sie sicher, dass dataSourceName mit dbname= endet. Oder der Datenbank-Adapter könnte abstürzen, wenn Sie Casdoor starten.

Ausführen​

Es gibt derzeit zwei Methoden zum Beginn, Sie können eine nach Ihrer eigenen Situation auswählen.

Entwicklungsmodus​

Backend​

Casdoor's Go Backend läuft standardmäßig am Port 8000. Sie können das Go-Backend mit dem folgenden Befehl starten:

go run main.go

Nachdem der Server erfolgreich läuft, können wir das Frontend-Teil starten.

Frontend​

Casdoor Frontend ist ein sehr klassisches Create-React-App (CRA) Projekt. Es läuft standardmäßig bei Port 7001 aus. Benutzen Sie die folgenden Befehle um das Frontend auszuführen:

cd web
yarn installieren
yarn start

Besuchen Sie: http://localhost:7001 in Ihrem Browser. Loggen Sie sich im Casdoor-Dashboard mit dem globalen Standard-Admin-Konto ein: eingebaut/admin

admin
123

Produktionsmodus​

Backend​

Build Casdoor Go Backend-Code in ausführbare Datei und starte ihn.

Für Linux:

go build
./casdoor

Für Windows:

go build
casdoor.exe

Frontend​

Build Casdoor frontend code in static resources (.html, .js, .css files):

cd web
yarn installieren
yarn build

Besuchen Sie: http://localhost:8000 in Ihrem Browser. Loggen Sie sich im Casdoor-Dashboard mit dem globalen Standard-Admin-Konto ein: eingebaut/admin

admin
123
Um einen anderen Port zu verwenden, bearbeiten Sie bitte conf/app.conf und ändern httpport, dann starten Sie das Go-Backend neu. :::

In dev Umgebung wird das Frontend von Garn ausgeführt in Port 7001 Wenn Sie also zur Casdoor Login-Seite gehen möchten, müssen Sie den Link "Casdoor" als http://localhost:7001 setzen.

In der prod Umgebung werden die Frontend Dateien zuerst von Garn Build erstellt und in Port 8000 geliefert, Also, wenn Sie zur Casdoor Login-Seite gehen möchten Sie müssen den Casdoor Link als http://SERVER_IP:8000 setzen (Wenn Sie Reverse Proxy verwenden, Sie müssen den Link als Ihre Domain setzen.

Nehmen Sie unser offizielles Forum Casnode als Beispiel:​

Kasnode verwendet Casdoor um die Authentifizierung zu behandeln.

When we are testing Casnode in dev environment, we set the serverUrl as http://localhost:7001, so when we test signin and signup functionality using Casdoor, it will go to localhost 7001 which is the Casdoor port.

Und wenn wir Casnode auf die prod Umgebung setzen, haben wir die serverUrl als https://door. asbin.com, damit Benutzer sich mit Casdoor anmelden oder sich anmelden können.
