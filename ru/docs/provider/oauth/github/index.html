Github

GitHub OAuth поддерживает как веб-поток приложений, так и поток устройств, пожалуйста, продолжайте чтение, чтобы получить данные OAuth.

Сначала посетите настройки разработчика GitHub , чтобы зарегистрировать новое приложение OAuth.

Затем заполните имя приложения, адрес домашней страницы, описание и URL обратного вызова авторизации.

::info правильно задать URL обратного вызова авторизации

В настройках GitHub OAuth URL обратного вызова авторизации должен быть обратным адресом вашего Касдорфа, и URL перенаправления в Casdoor должен быть вашим приложением callback url

Подробнее читайте конфигурацию приложения

:::

После регистрации GitHub OAuth вы можете создать ваш Client Secret сейчас!

Добавьте провайдера GitHub OAuth и заполните Client ID и Client Secret в вашем касстере

Теперь вы можете использовать GitHub как сторонний сервис для завершения аутентификации.

::caution

Tricks: Хотя мы не можем установить несколько редиректов в GitHub, поэтому в рабочей и производственной среде нам нужно установить различных поставщиков GitHub.

In development environment, Casdoor will only display the GitHub provider with "localhost" in its name. А в производственной среде на Casdoor будет отображаться только поставщик GitHub без "localhost" в его имени.
