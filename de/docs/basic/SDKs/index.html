Frontend & Backend SDK Setup

Einführung​

Als Service Provider (SP) unterstützt Casdoor zwei Authentifizierungsprotokolle:

  • OAuth 2.0 (OIDC)
  • SAML

Als Identity Provider (IdP) unterstützt Casdoor ein Authentifizierungsprotokoll:

  • OAuth 2.0 (OIDC)

Wenn Ihre Anwendung Casdoor als Identity Provider verwendet, werden diese über OIDC verbunden. Casdoor erfüllt den OIDC-Standard vollständig. Als Anwendungsentwickler können Sie jederzeit eine Standard-OIDC-Client-Bibliothek für Ihre Sprache verwenden, um sich mit Casdoor zu verbinden. Allerdings, wir bieten auch eine Reihe von Casdoor SDKs für populäre Sprachen zur Vereinfachung der OIDC-Interaktion und bieten einfach zu bedienende Wrapper für Casdoor Public API an.

Um Casdoor nutzen zu können, müssen Sie sowohl das Frontend SDK als auch das Backend SDK integrieren. Casdoor unterstützt die Authentifizierung sowohl für Web- als auch für mobile Anwendungen.

, wenn Ihre Web-Anwendung in Frontend und Backend getrennt entwickelt wurde, dann können Sie das Javascript SDK verwenden: casdoor-js-sdk um Casdoor in das Frontend zu integrieren. Wenn Ihre Webapplikation eine traditionelle Website ist, die von JSP oder PHP entwickelt wurde, können Sie nur die Backend SDKs verwenden. :::

Wie man einrichtet​

1. Initialisierung​

Der erste Schritt besteht darin, sich mit dem Casdoor-Server in Verbindung zu setzen. Sie müssen die Adresse des Casdoor-Servers angeben, die Credential ClientId und ClientSecret der Anwendung, das jwt-Geheimnis und der Name der Organisation, in der sich die Anwendung befindet, die alle String-Typen sind.

Ungefähr so:

Name (in Reihenfolge)MussBeschreibung
endpointJaCasdoor Server Url, wie http://localhost:8000
clientIdJaAnwendung.client_id
clientSecretJaAnwendung.client_secret
jwtSecretJaAus token_jwt_key.pem Datei lesen
organisationsnameJaAnwendung.Organisation

Die Funktionssignatur in Go ähnelt wie in anderen Sprachen:

func InitConfig(Endpunkt-String, Client-ID-String, clientSecret-String, jwtSecret-String, organizationName string)

Sie können sich auf die -Nutzung in Casnode beziehen,

var CasdoorEndpoint = "http://localhost:8000"
var ClientId = "541738959670d221d59d"
var ClientSecret = "66863369a64a5863827cf949bab70ed560ba24bf"
var JwtSecret = "CasdoorSecret"
var CasdoorOrganization = "casbin-forum"

func init() {
    auth. nitConfig(CasdoorEndpoint, ClientId, ClientSecret, JwtSecret, CasdoorOrganization)
}

Genaueres Verhalten kann sich auf die Quellcodes von casdoor-go-sdk beziehen.

2. SDK Zugriff auf Casdoor​

Bei vorheriger Konfiguration authConfigkann Casdoor SDK jetzt zum Casdoor Server wechseln, um Benutzer zu verifizieren Beachten Sie, dass SDK SetBasicAuth Methode in net/http Bibliothek verwendet, um ClientId und ClientSecret vom Casdoor Server zu erhalten und Benutzer in Ihrer Anwendung zu verifizieren.

Weitere Details entnehmen Sie bitte https://github.com/casdoor/casdoor-go-sdk/blob/master/auth/base.go#L42

3. Token und parsen​

Nach Ablauf der casdoor Verifizierung wird sie auf Ihre Anwendung mit Code und Status umgeleitet, wie http://forum.casbin.org?code=xxx&state=yyyy.

Ihre Webanwendung kann den -Code, Status abrufen und GetOAuthToken(Code, Status)aufrufen und dann jwt token parsen.

Siehe Verwendung in Casnode als Referenz:

code := c.Input().Get("code")
state := c.Input().Get("state")

token, err := auth. etOAuthToken(code, state)
if err != nil {
    panic(err)
}

claims, err := auth. arseJwtToken(token.AccessToken)
if err != nil {
    panic(err)
}

4. Setze Session in deiner App​

Sie müssen die Informationen überprüfen, die Sie im dritten Schritt geparst haben. Nachdem es korrekt ist (es kann modifiziert werden), beweist es, dass die Benutzerüberprüfung vorbei ist, und Sie können die Sitzung in Ihrer eigenen Anwendung einrichten.

An dieser Stelle ist das Benutzerlogin tatsächlich abgeschlossen.

Die Methode zur Session-Einstellung hängt stark von den Sprachen und Frameworks ab casnode verwendet das beego Framework und setzt die Session durch Verkapselungsmethode c. etSessionUser() , Quellcode.

Nutzung in Kasnode,

token, err := auth.GetOAuthToken(code, state)
if err != nil {
    panic(err)
}

claims, err := auth.ParseJwtToken(token. ccessToken)
if err != nil {
    panic(err)
}

claims. ccessToken = token.AccessToken
c.SetSessionUser(claims) // hier sehen

Das war's!

5. Interagiere mit der Benutzertabelle​

Möglicherweise müssen Sie sich nicht nur einloggen, sondern Sie erhalten auch Benutzerdaten, Aktualisierungsinformationen usw.

Das SDK bietet mehrere Schlüsselfunktionen und Sie müssen sie detaillierter anpassen oder ein Problem melden!

  • GetUser(name string), erhalte einen Benutzer mit Benutzernamen.
  • GetUsers(), holen Sie alle Benutzer.
  • UpdateUser(auth.User)/AddUser(auth.User)/DeleteUser(auth.User), schreibt Benutzer in die Datenbank.
  • CheckUserPassword(auth.User), überprüfen Sie das Passwort des Benutzers.
