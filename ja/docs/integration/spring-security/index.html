春のセキュリティ

CasdoorはIDPとしてOIDCプロトコルを使用して様々なアプリケーションを接続できます。 ここでは、アプリケーションへの接続にOIDCを使用する方法を例として、Spring Securityを使用します。

Step1. Casdoor を展開​

まず、Casdoorを展開する必要があります。

サーバーインストール については、Casdoor の公式ドキュメントを参照してください。

デプロイが完了したら、以下を確認する必要があります。

  • Casdoor サーバーは http://localhost:8000 で正常に実行されています。
  • お気に入りのブラウザを開き、 http://localhost:7001を参照してください。Casdoorのログインページが表示されます。
  • ログイン機能をテストするには、 管理者 と 123 を入力してください。

次に、以下の手順で、キャスドアベースのログインページを独自のアプリにすばやく実装できます。

Step2. Casdoorアプリケーションの設定​

  1. 既存のCasdoorアプリケーションを作成または使用します。
  2. リダイレクトURLを追加する
  3. 必要なプロバイダを追加し、その他の設定を補足します。

当然のことです。 アプリケーションの設定ページでの2つの値を取得できます: クライアントID と 上の図のような クライアントシークレット 。 次のステップで使用します

お気に入りのブラウザを開き、訪問: http://CASDOOR_HOSTNAME/.well-known/openid-configuration、CasdoorのOIDC設定が表示されます。

ステップ3. Spring Security の設定​

Spring SecurityはOIDCをネイティブにサポートしています。

Spring Security OAuth2 クライアントの設定をカスタマイズできます。

:::注意

特に <Client ID> などの独自のCasdoorインスタンスに設定を置き換える必要があります。

:::

  • application.yml
  • application.properties
spring:
  security:
    oauth2:
      client:
        registration:
          casdoor:
            client-id: <Client ID>
            client-secret: <Client Secret>
            scope: <Scope>
            authorization-grant-type: authorization_code
            redirect-uri: <Redirect URL>
        provider:
          casdoor:
            authorization-uri: http://CASDOOR_HOSTNAME:7001/login/oauth/authorize
            token-uri: http://CASDOOR_HOSTNAME:8000/api/login/oauth/access_token
            user-info-uri: http://CASDOOR_HOSTNAME:8000/api/get-account
            user-name-attribute: name
spring.security.oauth2.client.registration.casdoor.client-id=<Client ID>
spring.security.oauth2.client.registration.casdoor.client-secret=<Client Secret>
spring.security.oauth2.client.registration.casdoor.scope=<Scope>
spring.security.oauth2.client.registration.casdoor.authorization-grant-type=authorization_code
spring.security.oauth2.client.registration.casdoor.redirect-uri=<Redirect URL>

spring.security.oauth2.client.provider.casdoor.authorization-uri=http://CASDOOR_HOSTNAME:7001/login/oauth/authorize
spring.security.oauth2.client.provider.casdoor.token-uri=http://CASDOOR_HOSTNAME:8000/api/login/oauth/access_token
spring.security.oauth2.client.provider.casdoor.user-info-uri=http://CASDOOR_HOSTNAME:8000/api/get-account
spring.security.oauth2.client.provider.casdoor.user-name-attribute=name

コード内の ClientRegistration で設定をカスタマイズすることもできます。 マッピング はこちら

Step4. デモを始める​

  1. Spring Bootアプリケーションを作成できます。
  2. / と /login** を除くすべてのエンドポイントを保護する設定を追加できます。
@EnableWebSecurity
public class UiSecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.authorizeRequests().antMatchers("/", "/login**").permitAll().anyRequest().authenticated().and()
                .oauth2Login();

    }
}
  1. ログインするための素朴なページを追加できます。
<!DOCTYPE html>
<html lang="en">
<head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
<title>Spring OAuth Client Thymeleaf - 1</title>
<link rel="stylesheet"
    href="https://stackpath.bootstrapcdn.com/bootstrap/4.1.3/css/bootstrap.min.css" />
</head>
<body>
    <nav
        class="navbar navbar-expand-lg navbar-light bg-light shadow-sm p-3 mb-5">
        <a class="navbar-brand" th:href="@{/foos/}">Spring OAuth Client
            Thymeleaf - 1</a>
    </nav>
    <div class="container">
        <label>Welcome ! </label> <br /> <a th:href="@{/foos/}"
            class="btn btn-primary">ログイン</a>
    </div>
</body>
</html>

ユーザーが ログイン ボタンをクリックすると、 キャスドア にリダイレクトされます。

  1. 次に、保護されたリソースを定義することができます。 /foos と呼ばれるエンドポイントと表示用のウェブページをエクスポートすることができます。

データモデル

public class FooModel {
    private Long id;
    private String name;

    public FooModel(Long id, String name) {
        super();
        this.id = id;
        this.name = name;
    }
    public Long getId() {
        return id;
    }
    public void setId(Long id) {
        this.id = id;
    }
    public String getName() {
        return name;
    }
    public void setName(String name) {
        this.name = name;
    }
}

コントローラ

@Controller
public class FooClientController {
    @GetMapping("/foos")
    public String getFoos(Model model) {
        List<FooModel> foos = new ArrayList<>();
        foos.add(new FooModel(1L, "a"));
        foos.add(new FooModel(2L, "b"));
        foos.add(new FooModel(3L, "c"));
        model.addAttribute("foos", foos);
        return "foos";
    }
}

Web ページ

<!DOCTYPE html>
<html lang="en">
<head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
<title>Spring OAuth Client Thymeleaf - 1</title>
<link rel="stylesheet"
    href="https://stackpath.bootstrapcdn.com/bootstrap/4.1.3/css/bootstrap.min.css" />
</head>
<body>
    <nav
        class="navbar navbar-expand-lg navbar-light bg-light shadow-sm p-3 mb-5">
        <a class="navbar-brand" th:href="@{/foos/}">Spring OAuth Client
            Thymeleaf -1</a>
        <ul class="navbar-nav ml-auto">
            <li class="navbar-text">Hi, <span sec:authentication="name">preferred_username</span>&nbsp;&nbsp;&nbsp;
            </li>
        </ul>
    </nav>
    <div class="container">
        <h1>All Foos:</h1>
        <table class="table table-bordered table-striped">
            <thead>
                <tr>
                    <td>ID</td>
                    <td>Name</td>
                </tr>
            </thead>
            <tbody>
                <tr th:if="${foos.empty}">
                    <td colspan="4">No foos</td>
                </tr>
                <tr th:each="foo : ${foos}">
                    <td><span th:text="${foo.id}"> ID </span></td>
                    <td><span th:text="${foo.name}"> Name </span></td>
                </tr>
            </tbody>
        </table>
    </div>
</body>
</html>

:::注意

すべての Web ページテンプレートは resources/templates の下に置く必要があります。

:::

ステップ5. デモをお試しください！​

まず、お気に入りのブラウザを開き、 /foosに直接アクセスしてみることができます。 それは自動的にカジノのログインページにリダイレクトされます。 こちらまたはルートページからログインできます。

ルートページにアクセスした場合は、

ログイン ボタンをクリックすると、ページは casdoor のログインページにリダイレクトされます。

ログインすると、ページは /foos にリダイレクトされます。
