Aliyun IDaaS

Aliyun IDaaSでSAMLアプリケーションを作成​

Aliyun 管理コンソールにログインし、Application Indentity サービス(IDentity-as-a-Service, IDaaS)を検索します。

EIAM インスタンス リスト をクリックして、無料バージョンを開きます。

インスタンスが作成され、開いた後に自動的に実行されます。 インスタンス名または 管理 ボタンをクリックして IDaaS 管理コンソールを入力します。

IDaaS管理コンソールを入力したら、 Add Applicationをクリックし、 SAMLを検索し、 Add Application をクリックします。

Add SigningKey をクリックします。

必要な情報をすべて入力し、提出してください。

追加された署名キーを選択します。

以下のように必要な情報をすべて入力し、提出してください。

  • IDP IdentityId: Casdoor の Issuer URL と同じにしてください。
  • SP エンティティID & SP ACS URL(SSO Location):必要なものを入力します。 Casdoorの設定が完了したら、変更する必要があります。
  • アサーション属性: ユーザー名として直接入力します。
  • アカウントの関連モード: アカウントの関連

アカウント認証 & 関連​

appplicationが正常に追加されると、認証プロンプトが表示されます。 今すぐ認証しないで、アカウントを追加してから認証します。

組織とグループ に移動し、 新規アカウントをクリックします。

必要な情報をすべて入力し、提出してください。

Application Authorizationに移動し、認証するアカウントを選択し、 Save をクリックします。

アプリケーション リストに移動し、 アプリケーション サブアカウントを表示をクリックし、 アカウントの関連付けを追加 をクリックします。

関連付けが必要なプライマリアカウントとサブアカウントを入力し、 保存 をクリックします。

プライマリアカウントは IDaaS に存在し、サブアカウントは Casdoor のユーザの ID です。

IDaaSメタデータをエクスポート​

Application Listに移動し、 View Application Details をクリックし、 Export IDaaS SAML Metadadata をクリックします。

Casdoorで設定​

Casdoorに新しいプロバイダを作成します。

SAMLとしてカテゴリを選択し、 Aliyun IdaaS と入力します。 メタデータの内容をコピーし、 メタデータ 入力に貼り付けます。 The values of Endpoint, IdP and Issuer URL will be generated automatically after clicking the Parse button.

SP ACS URL と SP ID をコピーし、 Save ボタンをクリックします。

Casdoorで設定したいアプリケーションを編集します。 追加されたプロバイダを選択し、ボタン 保存をクリックします。

Aliyun IDaaSでSAMLアプリケーションを変更​

アプリケーションを無効にし、 Modify Application をクリックします。

Casdoorにコピーされたコンテンツを SP ID と SP ACS URL(SSO Location) に記入します。 提出してアプリケーションを有効にします。

効果を検証する​

設定したアプリケーションに移動すると、ログインページにアイコンが表示されます。

アイコンをクリックし、Aliyun IDaaSログインページにジャンプし、認証後にCasdoorに正常にログインします。
