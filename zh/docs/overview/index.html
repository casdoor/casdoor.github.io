Overview

Casdoor 是一个UI优先的基于 OAuth 2.0 / OIDC 的集中的单点登录(SSO)身份验证平台。

Casdoor 可为网页UI和应用用户登录请求提供服务。

Casdoor 特性：​

  1. 前后端分离的架构，使用Go语言开发，Cas door支持高并发，提供基于网页的用户管理UI并且支持多语言(中文和英文)。

  2. Casdoor 支持使用 Github、Google、QQ、微信等第三方应用程序登录，并且支持使用插件来拓展第三方登录。

  3. 使用基于 Casbin 的授权管理，Casdoor 支持 ACL，RBAC，ABAC，RESTful 访问控制模型。

  4. 支持手机验证码、邮件验证码和找回密码功能。

  5. 审查和记录访问日志。

  6. 支持使用阿里云、腾讯云、七牛云提供的图片CDN云存储。

  7. 个性化的注册、登录和忘记密码页面。

  8. Casdoor 支持使用数据库同步的方法来与现有系统集成，用户可以平滑过渡到 Casdoor。

  9. Casdoor 支持 MySQL、PostgreSQL、SQL Server 等主流数据库，并且支持使用插件来拓展其它数据库。

工作原理:​

步骤0 (前置知识)​

  1. Cassdoor的授权程序建立在OAuth 2的基础上。 因此，强烈建议简单了解OAuth 2.0 是否起作用。 An introduction of OAuth 2.0.

步骤 1 (授权请求)​

您的应用 (可以是网站或其他) 应该以这种格式 endpoint/login/oauth/authorize?client_id=xxx&response_type=code&redirect_uri=xxx&scope=read&state=xxx. 在 URL 中用 Cassdoor 的主机 URL 替换 endpoint ，并用您自己的信息替换 xxx。

:::提示

如何填充 xxx ？

  • 对于 client_id: 您可以在每个应用程序下找到它

  • 对于 重定向_uri: 你应该将此设置为你自己的应用程序回调URL, 通过这个信息，Casdoor 可以知道在授权后如何回发信息

  • 对于 状态: 您应该用您的应用名称填写这个内容

:::

应用程序将通知用户： "嘿，现在我需要一些资源，我需要你的许可才能拿这些资源。 您要转到这个URL，填写您的用户名和密码？

使用正确的组合URL，您的应用将会向此 URL 发起请求， 然后授权请求 已完成。

步骤 2 (授权认证)​

此步骤是直接的：用户被重定向到上面的URL，用户应该看到来自Casdoor的登录页面。
通过在登录表单中输入正确的用户名和凭据，Casdoor现在知道用户的身份。 并且发送两个关键词： 代码 和 状态 返回第 1 步中设置的回调URL。

用户打开网址并向Cassdoor提供凭据。 Casdoor会说： "好~ 这是我在数据库中知道的用户 (授权应用程序获取 代码 和 状态)。 然后我将使用回调URL发送 代码 和 状态 返回应用 (redirect_uri)"

在这两个关键词回到您的申请中，现在已经授予应用认证，因此 授权 已经完成。

:::提示

Cassdoor 也提供第三方登录。在这种情况下，您将不会看到凭据输入页面，而是会看到第三方提供商列表。 您可以使用这些提供商登录到您的应用，而Casdoor是中间层(中间件)。

:::

步骤 3 (授权认证)​

在这个步骤中，您的应用程序已经有了来自第2步的代码，并且它将会告诉Casdoor： "嘿, 现在用户同意给我 代码, 您想检查这个 代码 并给我 access_token?

步骤 4 (访问令牌)​

在这个步骤中，Casdoor 会通知应用程序： "此 代码 似乎是合法的，你必须是正确的应用程序。 这是 访问令牌。
使用相关的 代码, Casdoor 知道它是一个授权的应用(第二步中正确用户给予的授权)，试图获取 access_token (稍后将被用来获取更有用的信息)。

步骤 5 (访问令牌)​

在这个步骤中，您的应用： "获得了新的 访问令牌现在可以使用它从 资源服务器获得更多有用的信息！

您的应用转向 资源服务器: 检查这个 访问令牌 我从Casdoor得到了访问令牌，你想看看这是否与Casdoor的一致？

步骤 6 (受保护资源)​

Resource Server 回来通知您的应用程序： " 它和我在 Cassdoor 中的那个一样。 Casdoor说谁拥有这个 access_token 谁可以拥有这些 受保护的资源 现在，你可以拿走了！”

这基本上就是Casdoor 如何与您的应用协作。

:::提示

Cassdoor 可以同时运行 认证服务器 和 资源服务器 ，也就是说： Casdoor 授权应用从Casdoor的数据库获取资源 (通常是当前登录用户的信息)。

:::
