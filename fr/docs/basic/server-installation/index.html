Installation du serveur

Exigences​

Système d'exploitation​

Tous les systèmes d'exploitation (Linux, Windows et macOS) sont pris en charge.

Environnement​

  • Go 1.6+
  • Node.js LTS (16 ou 14)
  • Yarn 1.x
info

Nous vous suggérons fortement d'utiliser Yarn 1. pour exécuter & construire Casdoor frontend, utiliser NPM peut causer des problèmes de style d'interface d'interface d'interface utilisateur voir plus de détails à : casdoor#294

Base de données​

Casdoor utilise XORM pour parler à la base de données. Basé sur le support de Xorm Drivers, les bases de données actuellement prises en charge sont :

  • MySQL
  • MariaDB
  • PostgreSQL
  • Serveur SQL
  • Oracle
  • SQLite 3
  • TiDB

Télécharger​

Le code source de Casdoor est hébergé sur GitHub : https://github.com/casdoor/casdoor . Le code du backend Go et le code du frontend React sont tous deux à l'intérieur du dépôt unique.

NomLibelléLangueCode source
FrontendInterface web pour CasdoorJavaScript + Reacthttps://github.com/casdoor/casdoor/tree/master/web
BackendBackend API RESTful pour CasdoorGolang + Beego + XORMhttps://github.com/casdoor/casdoor

Casdoor prend en charge Go Modules. Pour télécharger le code, vous pouvez simplement cloner le code via git:

cd path/to/folder
git clone https://github.com/casdoor/casdoor

Configuration​

Casdoor peut être configuré via un seul fichier : conf/app.conf, dont le contenu par défaut est :

appname = casdoor
httpport = 8000
runmode = dev
SessionOn = true
copyrequestbody = true
driverName = mysql
dataSourceName = root:123456@tcp(localhost:3306)/
dbName = casdoor
redisEndpoint =
defaultStorageProvider =
isCloudIntranet = false
authState = "casdoor"
httpProxy = "127. .0.1:10808"
verificationCodeTimeout = 10
initScore = 2000
logPostOnly = true
origine = "https://door.casbin.com"
  • appname est le nom de l'application, qui n'a actuellement aucune utilité pratique
  • httpport est le port sur lequel votre application back-end est en train d'écouter
  • runmode est dev ou prod
  • SessionOn décide d'activer la session, utilisée par défaut.
  • driverName, dataSourceName et dbName sont introduits avant, veuillez consulter Config.
  • useProxy a défini le port du proxy, car nous avons des services liés à Google, qui seront restreints par le réseau dans certaines zones
  • verificationCodeTimeout définit le temps d'expiration du code de vérification. Après l'expiration, l'utilisateur doit l'obtenir à nouveau

Malgré tous les champs configurables, en tant que débutant, vous devez seulement modifier deux éléments : driverName et dataSourceName en fonction de votre base de données. Cette base de données sera utilisée par Casdoor pour stocker toutes les données, y compris les utilisateurs, les organisations, les applications, etc.

  • Configuration de la base de données (MySQL):

    Casdoor stockera les informations de ses utilisateurs, nœuds et sujets dans une base de données MySQL nommée : casdoor, la créera si elle n'existe pas. La chaîne de connexion DB peut être spécifiée à : https://github.com/casdoor/casdoor/blob/master/conf/app.conf

    driverName = mysql
    dataSourceName = root:123456@tcp(localhost:3306)/
    dbName = casdoor

  • Configuration de la base de données (Postgres):

    Puisque nous devons choisir une base de données lors de l'ouverture de Postgres avec xorm, vous devez préparer une base de données manuellement avant d'exécuter Casdoor.

    Supposons que vous avez déjà préparé une base de données appelée casdoor, alors vous devriez spécifier app.conf comme ceci :

    driverName = postgres
    dataSourceName = "user=postgres password=xxx sslmode=disable dbname="
    dbName = casdoor

    Veuillez noter : Vous pouvez ajouter des paramètres Postgres dans dataSourceName, mais assurez-vous que dataSourceName se termine par dbname=. Ou l'adaptateur de base de données peut planter lorsque vous lancez Casdoor.

Exécuter​

Il y a actuellement deux méthodes pour commencer, vous pouvez en choisir une selon votre propre situation.

Mode développement​

Backend​

Le backend de Casdoor's Go s'exécute par défaut au port 8000. Vous pouvez démarrer le backend Go avec la commande suivante :

exécuter main.go

Une fois que le serveur est en cours d'exécution, nous pouvons démarrer la partie frontend.

Frontend​

Le frontend de Casdoor est un projet très classique Create-React-App (CRA). Il tourne au port 7001 par défaut. Utilisez les commandes suivantes pour exécuter le frontend :

cd web
yarn install
yarn start

Visiter : http://localhost:7001 dans votre navigateur. Connectez-vous au tableau de bord de Casdoor avec le compte d'administration global par défaut : intégré/admin

admin
123

Mode production​

Backend​

Construisez le code d'arrière-plan Casdoor Go dans l'exécutable et démarrez-le.

Pour Linux :

construisez
./casdoor

Pour Windows :

allez construire
casdoor.exe

Frontend​

Construire du code frontend Casdoor en ressources statiques (fichiers .html, .js, .cs):

cd web
yarn install
yarn build

Visiter : http://localhost:8000 dans votre navigateur. Connectez-vous au tableau de bord de Casdoor avec le compte d'administration global par défaut : intégré/admin

admin
123
Pour utiliser un autre port, veuillez éditer conf/app.conf et modifier httpport, puis redémarrer le Go backend. :::

Dans l'environnement de développement dev , le frontend est exécuté par yarn run dans le port 7001, donc si vous voulez aller à la page de connexion de Casdoor , vous devez définir le lien Casdoor comme http://localhost:7001.

Dans l'environnement prod , les fichiers frontend sont d'abord construits par yarn build et servis dans le port 8000, donc si vous voulez aller à la page de connexion de Casdoor vous devez définir le lien Casdoor comme http://SERVER_IP:8000 (Si vous utilisez un proxy inverse, vous devez définir le lien comme votre domaine).

Prenons l'exemple de notre forum officiel Casnode :​

Casnode utilise Casdoor pour gérer l'authentification.

Lorsque nous testons Casnode dans l'environnement dev , nous définissons l'adresse serverUrl comme http://localhost:7001, de sorte que lorsque nous testons la fonctionnalité de connexion et d'inscription en utilisant Casdoor, il ira sur localhost 7001 qui est le port Casporto.

Et lorsque nous mettons Casnode dans l'environnement prod , nous définissons le serverUrl comme https://door. asbin.com, afin que les utilisateurs puissent se connecter ou s'inscrire en utilisant Casdoor.
