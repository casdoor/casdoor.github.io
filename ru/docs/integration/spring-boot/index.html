Весенний сапог

casdoor-spring-boot-example - пример использования casdoor-java-sdk в проекте SpringBoot. Мы покажем вам шаги ниже.

Шаг1. Развернуть корпус​

Во-первых, следует развернуть корпус.

Вы можете обратиться к официальной документации Casdoor для Server Installation.

После успешного развертывания, вы должны обеспечить:

  • Сервер Casdoor успешно запущен на http://localhost:8000.
  • Откройте свой любимый браузер и посетите http://localhost:7001, вы увидите страницу входа в Casdoor.
  • Ввод администратора и 123 для проверки функции входа в систему отлично.

Затем вы можете быстро реализовать страницу входа на кассе в вашем собственном приложении со следующими шагами.

Шаг2. Импорт casdoor-java-sdk​

Вы можете импортировать касдор-ява-сдк с лабиринтом или градацией.

  • Maven
  • Gradle
<dependency>
    <groupId>org.casbin</groupId>
    <artifactId>casdoor-java-sdk</artifactId>
    <version>1.x.y</version>
</dependency>
implementation 'org.casbin:casdoor-java-sdk:1.x.y'

Шаг3. Конфигурация Init​

Инициализация требует 6 параметров, которые являются всеми типами строк. | Название (в порядке) | Должен | Описание | | -------------------- | ------ | -------------------------------------------------------------- | | endpoint | Да | Адрес сервера Casdoor Server, например http://localhost:8000 | | clientId | Да | Приложение.client_id | | clientSecret | Да | Application.client_secret | | jwtSecret | Да | То же самое, что и секрет JWT Кастрюля. | | название организации | Да | Приложение.организация | | appationName | Да | Application.name | Для инициализации вы можете использовать свойства Java или YAML файлы.

  • Properties
  • YML
casdoor.endpoint = http://localhost:8000
casdoor.clientId = 874e3e05e58d50148c65
casdoor.clientSecret = 41510b84c7267ad2e4d2b51096b7f11dc9c5fdc8
casdoor.jwtSecret = CasdoorSecret
casdoor.organizationName = built-in
casdoor.applicationName = app-built-in
casdoor:
  endpoint: http://localhost:8000
  client-id: 874e3e05e58d50148c65
  client-secret: 41510b84c7267ad2e4d2b51096b7f11dc9c5fdc8
  jwt-secret: CasdoorSecret
  organization-name: built-in
  application-name: app-built-in

::caution Вы должны заменить конфигурацию своим собственным экземпляром Casdoor clientId и clientSecret. ::: Затем создайте класс CasdoorSdkProperties , объявьте переменные члена, соответствующие конфигурации и предоставьте getter и setter.

@Data
@Component
@ConfigurationProperties(prefix = "casdoor")
public class CasdoorSdkProperties {
    private String endpoint;
    private String clientId;
    private String clientSecret;
    private String jwtSecret;
    private String organizationName;
    private String applicationName;
}

Создайте класс CasdoorSdkConfig , init CasdoorConfig с экземпляром CasdoorSdkProperties.

@Configuration
public class CasdoorSdkConfig {

    @Resource
    private CasdoorSdkProperties casdoorSdkProperties;

    @Bean
    public CasdoorConfig getCasdoorConfig() {
        return new CasdoorConfig(
                casdoorSdkProperties.getEndpoint(),
                casdoorSdkProperties.getClientId(),
                casdoorSdkProperties.getClientSecret(),
                casdoorSdkProperties.getJwtSecret(),
                casdoorSdkProperties.getOrganizationName(),
                casdoorSdkProperties.getApplicationName()
        );
    }
}

При запуске приложения SpringBoot, конфигурация в свойствах Java или YAML автоматически вводится в переменные членов в классе CasdoorSdkProperties.

Поэтому вы можете использовать метод getCasdoorConfig в классе CasdoorSdkConfig , чтобы получить экземпляр CasdoorConfig где угодно.

Шаг4. Перенаправить на страницу входа​

Когда вам нужна аутентификация, которая имеет доступ к вашему приложению, вы можете отправить целевой url и перенаправить на страницу входа, предоставленную Casdoor.

Пожалуйста, убедитесь, что вы добавили обратный адрес (например, http://localhost:8080/login) в настройках приложения.

@RequestMapping("toLogin")
public String toLogin() {
    CasdoorConfig casdoorConfig = casdoorSdkConfig.getCasdoorConfig();
    String targetUrl = String.format("%s/login/oauth/authorize?client_id=%s&response_type=code&redirect_uri=%s&scope=read&state=%s",
            "http://localhost:7001", casdoorConfig.getClientId(),
            "http://localhost:8080/login", casdoorConfig.getApplicationName());
    return "redirect:" + targetUrl;
}

Шаг5. Получить токен и разобрать​

После прохождения проверки Casdoor он будет перенаправлен в ваше приложение с кодом и состоянием.

Вы можете получить код и вызвать метод getOAuthToken , а затем разобрать токен jwt.

CasdoorUser содержит базовую информацию о пользователе, предоставленную Casdoor, можно использовать его в качестве ключевого слова для установки сеанса в приложении.

@RequestMapping("login")
public String login(String code, String state, HttpServletRequest request) {
    CasdoorAuthService casdoorAuthService = new CasdoorAuthService(casdoorSdkConfig.getCasdoorConfig());
    String token = "";
    CasdoorUser user = null;
    try {
        token = casdoorAuthService.getOAuthToken(code, state);
        user = casdoorAuthService.parseJwtToken(token);
    } catch (OAuthSystemException | OAuthProblemException | ParseException | InvocationTargetException | IllegalAccessException e) {
        e.printStackTrace();
    }
    HttpSession session = request.getSession();
    session.setAttribute("casdoorUser", user);
    return "redirect:/";
}

Step6. Сервис пользователя​

CasdoorUserService поддерживает основные операции пользователя, такие как:

  • GetUser(name string), получить одного пользователя по имени пользователя.
  • GetUsers(), собрать всех пользователей.
  • UpdateUser(auth.User)/AddUser(auth.User)/DeleteUser(auth.User), запись пользователя в базу данных.
