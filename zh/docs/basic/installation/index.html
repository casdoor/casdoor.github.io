Set up Casdoor

安装#

安装 Casdoor 非常简单，你可以使用go get来获取：

go get github.com/casbin/casdoor

或者使用git clone

git clone https://github.com/casbin/casdoor

配置#

所有的配置文件都在 /conf 目录中，包括 app.conf 和 oss.conf。 前者存储应用程序相关的信息，而后者仅存储 OSS 相关的配置。 所有的设置都可以通过修改这两个文件来进行。

在第一次使用的时候，你不需要搞清楚所有配置选项，这些东西将在后面介绍。 现在你只需要配置几个简单的选项来在自己的电脑上运行 casdoor。

无论如何，数据库信息应该最先被配置。 Casdoor 使用XORM 框架来连接数据库，因此，所有XORM支持的数据库都可以使用。

  • 设置数据库 (MySQL)：

    Casdoor 把用户、节点还有主题信息保存在MySQL数据库的casdoor库中，如果这个数据库不存在就会自动创建一个。 数据库连接字符串可以在 https://github.com/casbin/casdoor/blob/master/conf/app.conf 这个文件上指定。

    driverName = mysqldataSourceName = root:123@tcp(localhost:3306)/dbName = casdoor

  • 设置数据库 (Postgres)：

    由于在使用xorm打开 Postgres 时必须选中一个数据库，所以你应该在启动 Casdoor 前手动准备好数据库。

    假设你已经创建了一个叫做 casdoor 的数据库，然后你应该把 app.conf 修改为这样：

    driverName = postgresdataSourceName = "user=postgres password=xxx sslmode=disable dbname="dbName = casdoor

    注意： 你可以通过添加 dataSourceName 参数来添加 Postgres 参数，但是请确保 dataSourceName 以 dbname= 结尾。 否则数据库适配器可能在你启动 Casdoor 的时候崩溃。

运行#

当你完成了上述简单的设置步骤后，你可以启动 Casdoor。 Casdoor 是前后端分离的架构。 前端使用了 javascript 和 React，后端使用了beego 和 xorm。

后端#

go run main.go

或者先构建再运行：

go build && ./main

前端#

Casdoor 使用 npm 和 yarn 作为包管理器，你可以使用其中任意一个。 打开一个新的终端并且输入：

cd web
## npmnpm installnpm run start
## yarnyarn installyarn run start

很好，在你喜欢的浏览器里访问 http://127.0.0.1:7001 ，如果你能看见一个 Casdoor 的登录界面，就说明所有东西都没问题。

详细信息#

  • Github 角标

    我们在右上角添加了Github的角标，可以链接到你的Github仓库地址。 你可以通过设置 ShowGithubCorner 来隐藏它。

    配置如下：

    export const ShowGithubCorner = true
export const GithubRepo = "https://github.com/casbin/casdoor" //你的 github 仓库

  • OSS 设置

    我们使用 OSS 来存储和显示用户的头像。 你必须修改文件 conf/oss.conf 来告诉后端 OSS 相关的信息。 我们支持阿里云([aliyun]) 和亚马逊S3([s3]) 作为 OSS 提供商。

    [provider]accessId = idaccessKey = keybucket = bucketendpoint = endpoint

    请正确填写这个配置文件，否则头像服务没有办法正常工作！

  • 应用程序设置

    conf/app.conf 存储的是 casdoor 的重要设置，有助于你更好的使用 Casdoor 。

    appname = casdoorhttpport = 8000runmode = devSessionOn = truecopyrequestbody = truedriverName = mysqldataSourceName = root:123@tcp(localhost:3306)/dbName = casdoorauthState = "casdoor"useProxy = falseverificationCodeTimeout = 10
    • appname 是应用程序的名称，目前来说并没有实际用途。
    • httpport 是后端程序监听的端口。
    • rundev 用于选择开发环境 dev 或者生产环境 prod。
    • SessionOn 控制是否启用会话，默认为使用。
    • driverName, dataSourceName 以及 dbName 前面已经介绍过了，故不在此赘述。
    • useProxy 设置代理端口，因为我们使用了在某些地区由于网络不可用的谷歌相关的服务。
    • verificationCodeTimeout 设置验证码超时时间。 超时之后用户需要再次获取验证码。

  • If casdoor needs to be publicly accessible(possess public IP), also need to modify web/src/auth/Auth.js:

    export let authConfig = {  serverUrl: "http://example.com", // your Casdoor URL, like the official one: https://door.casbin.com, or http:1.1.1.1:8000  ...}
