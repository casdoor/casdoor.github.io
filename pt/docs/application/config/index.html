Configuração da Aplicação

Depois de implantar o Casdoor no seu servidor e configurar sua organização, agora você pode implantar suas aplicações!

Vamos ver como configurar a autenticação da sua aplicação usando o Casdoor!

nota

Por exemplo, quero configurar meu Fórum usando Casnode.

Crio minha aplicação e preencho algumas configurações necessárias.

Seleciono a organização que criei para que os usuários desta organização possam usar esta aplicação.

Como esta organização é chamada my_organization, eu a escolho no menu suspenso.

A seguir, quero que meus usuários possam usar o Casdoor para autenticação quando se cadastrarem. Então, eu preencho a URL de redirecionamento aqui como https://your-site-url.com/callback.

cuidado

Observe que a callback URL na aplicação provedora deve ser a URL de callback do Casdoor, e a Redirect URL no Casdoor deve ser a URL de callback do seu site.

Entendimento Adicional​

Para fazer o processo de autenticação funcionar, os passos detalhados são os seguintes:

  1. Os usuários enviam uma solicitação ao Casdoor.
  2. O Casdoor usa o Client ID e o Client Secret para se autenticar com o GitHub, Google ou outros provedores.
  3. Se a autenticação for bem-sucedida, o GitHub chama de volta ao Casdoor para notificar sobre a autenticação bem-sucedida. Portanto, a URL de callback de autorização do GitHub deve ser a URL de callback do seu Casdoor, que é http://your-casdoor-url.com/callback.
  4. O Casdoor então informa a aplicação sobre o sucesso da autenticação. Isso significa que a URL de callback do Casdoor deve ser a URL de callback da sua aplicação, que é http://your-site-url.com/callback.
dica

Se você quiser fazer uma configuração mais personalizada dos métodos de login da aplicação, como desativar um determinado método de login ou desligar um determinado método de login, você pode consultar o Métodos de Login
